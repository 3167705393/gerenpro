# 企业级智能拓客平台与亿级底层数据架构开发

## 项目背景

在B2B销售场景中，销售团队面临着"找客户难、找对客户更难"的核心痛点。传统获客方式依赖人工搜索、行业展会或购买昂贵的第三方数据服务，存在效率低、成本高、数据质量参差不齐等问题。本项目旨在构建一个智能化的企业获客助手，通过自然语言交互实现精准的客户线索挖掘。

## 技术架构

```
用户输入 → 安全审查层 → 意图重写器 → 意图分类器 → 拓客子工作流
                ↓                              ↓
         代码+LLM双重防护              信息提取节点
                                              ↓
                                    MCP工具调用(自研)
                                              ↓
                                    SQL Server亿级数据查询
                                              ↓
                                    飞书Bitable监控埋点
```

## 核心技术实现

### 1. 多维特征提取与安全风控

**技术难点**: 如何从模糊的自然语言中精准提取结构化查询条件，同时防止恶意请求和非法查询。

**解决方案**:
- 设计基于 **qwen2.5-72b-instruct** 的大模型信息抽取节点，从自然语言中精准提取地域、行业、应用场景及企业类型四个核心维度
- 内置多层安全合规约束：
  - **非法行业拦截**: 识别走私、毒品、洗钱等关键词，自动置空返回
  - **虚构地域校验**: 验证输入地域是否为真实行政区域，防止幻觉数据产生
  - **数值强制阻断**: 单次查询数量上限锁定为50家，防止资源滥用
- 采用结构化输出确保数据格式一致性

```python
# 安全约束示例逻辑
if industry in ["走私", "毒品", "洗钱", "发票买卖"]:
    return {"industry": "", "application": ""}  # 置空拦截

if region not in VALID_REGIONS:  # 地域白名单校验
    return {"region": ""}  # 防止"月球"、"霍格沃茨"等虚构输入
```

### 2. 跨节点任务调度与监控

**技术难点**: 如何实现主Agent与拓客子工作流的解耦调用，并建立全链路监控体系。

**解决方案**:
- 通过 **local_chatflow_invoker** 插件实现主Agent与拓客子工作流的无缝解耦调用，支持会话上下文保持
- 接入飞书 **Bitable (HTTP Request)** 实现全链路自动化埋点：
  - 初始问题记录
  - 意图补全问题追踪
  - 输出结果归档
  - 为后续数据分析提供完整链路

### 3. 拓客架构演进与自研MCP工具（核心亮点）

**技术难点**: 第三方API成本高昂（单次查询费用约0.5-2元），且数据覆盖度有限，无法满足高频、定制化的企业查询需求。

**解决方案**:
独立承担超**1亿条（100GB+）**多源异构CSV工商数据的全链路清洗与入库工程：

**数据清洗层**:
```python
import polars as pl

# 使用Polars高效处理脏数据
def clean_company_data(raw_df):
    return (
        raw_df
        .filter(pl.col("企业状态") == "存续")  # 过滤注销企业
        .with_columns([
            pl.col("统一社会信用代码").str.strip(),  # 清洗空格
            pl.col("注册资本").str.extract(r"(\d+)").cast(pl.Int64),  # 提取数值
            pl.col("成立日期").str.to_date("%Y-%m-%d")  # 日期标准化
        ])
        .unique(subset=["统一社会信用代码"])  # 去重
    )
```

**入库优化**:
- 优化SQL Server批量入库机制，采用批量插入代替逐条插入，入库效率提升**10倍以上**
- 建立合理的索引策略（地域、行业、企业类型联合索引）

**MCP工具封装**:
```python
# 封装为自定义MCP工具，实现大模型原生调用
@mcp.tool()
def query_companies(region: str, industry: str, limit: int = 10) -> list:
    """
    查询符合条件的企业列表
    Args:
        region: 地域（如"广东"、"汕头"）
        industry: 行业（如"塑料制造"、"化工"）
        limit: 返回数量（最大50）
    Returns:
        企业信息列表
    """
    query = """
        SELECT 企业名称, 统一社会信用代码, 注册资本, 成立日期, 经营范围
        FROM enterprises
        WHERE 地区 LIKE ? AND 行业 LIKE ? AND 企业状态 = '存续'
        ORDER BY 注册资本 DESC
        LIMIT ?
    """
    return execute_query(query, [f"%{region}%", f"%{industry}%", min(limit, 50)])
```
