# 全国企业工商信息数据处理项目说明

## 项目概述

本项目旨在处理来自全国各地的企业工商注册信息数据，通过对大量Excel格式的数据文件进行清洗、标准化和转换，最终将数据导入SQL Server数据库中，形成统一的企业信息数据库。该项目解决了原始数据格式不统一、数据质量参差不齐的问题，实现了大规模企业数据的自动化处理。

## 项目目标

1. **数据清洗**：统一不同来源、不同格式的企业工商信息数据
2. **格式标准化**：将各种列名映射到统一的标准列名体系
3. **数据转换**：将Excel文件转换为易于处理的CSV格式
4. **数据导入**：将清洗后的数据高效导入SQL Server数据库
5. **数据质量管理**：通过多种校验机制确保数据准确性

## 项目架构与流程

### 1. 数据预处理阶段

#### 1.1 文件转换
- **转换2.0（最稳定版本，速度慢流式写入不占内存）.py**
  - 使用流式写入方式，内存占用极低
  - 适用于大批量Excel文件转换
  - 支持.xls和.xlsx格式
  
- **转换3.0（写的飞快，但是兼容性差，对xls兼容不好，会失败）.py**
  - 高速转换，但兼容性较差
  - 采用智能降级策略处理不同格式文件
  - 使用xlsx2csv和pandas的calamine引擎双重保障

#### 1.2 数据统计分析
- **统计列名 csv.py**：统计所有CSV文件中的列名出现频率
- **统计列名带文件名.py**：详细记录每个列名出现在哪些文件中
- **查询 每个文件csv行数.py**：统计每个CSV文件的数据行数

### 2. 数据清洗阶段

#### 2.1 核心清洗脚本
- **清洗csv文件针对 兼容 列名不同的格式 针对省份地区文件.py**
  - 核心功能：将不同列名映射到统一标准列名
  - 处理不同省份、地区数据格式差异
  - 多进程并行处理，提高效率
  - 输出Tab分隔的CSV文件，避免引号问题

- **转换后清洗数据普遍（针对全国工商信息文件进行清洗）.py**
  - 对转换后的数据进行二次清洗
  - 确保数据质量和一致性

#### 2.2 列名映射规则
项目定义了详细的列名映射规则，例如：
```python
COLUMN_MAPPING = {

    # 行业映射逻辑
```

### 3. 数据导入阶段

#### 3.1 本地数据库导入
- **多核心导入本地版数据库.py**
  - 多进程并行导入，充分利用CPU资源
  - 使用SQLAlchemy连接SQL Server数据库
  - 支持断点续传和错误处理

#### 3.2 服务器数据库导入
- **多核心导入服务器 等待30分钟后运行.py**
  - 专为服务器环境优化
  - 延迟启动避免系统负载过高
  - 适配远程数据库连接

#### 3.3 单线程导入（备用）
- **导入数据库.py**
  - 简化的单线程导入脚本
  - 适用于小批量数据或调试

## 技术特点

### 1. 高性能处理
- 使用Polars进行高性能数据处理
- 多进程并行处理充分利用多核CPU
- 流式写入减少内存占用
- Tab分隔符避免CSV引号解析问题

### 2. 数据质量控制
- 经营状态过滤（仅保留"正常"、"在业"、"开业"、"存续"状态）
- 数据长度截断防止数据库字段溢出
- 特殊字符清洗（去除制表符、换行符、引号等）
- 注册资金数字化处理和评分计算

### 3. 容错与兼容性
- 多种编码格式兼容（UTF-8、GBK等）
- Excel格式智能识别和处理
- 错误文件自动跳过和记录
- 断点续传机制

## 数据库设计

### 目标表结构（CompanyData_New）
项目将数据导入到具有以下关键字段的表中：
- 企业名称
- 统一社会信用代码
- 企业类型
- 经营状态
- 所属行业
- 成立日期
- 法定代表人
- 注册资本
- 注册地址
- 联系电话
- 网站链接
- 所属省份/城市/区县
- 注册资金数字（数值化）
- StaticScore（企业评分）

## 使用说明

### 1. 环境准备
- Python 3.8+
- 安装依赖包：polars, pandas, sqlalchemy, tqdm, xlsx2csv等
- 配置SQL Server数据库连接
- 安装ODBC Driver 17 for SQL Server

### 2. 执行流程
1. **数据转换**：运行转换脚本将Excel文件转换为CSV
2. **数据清洗**：运行清洗脚本统一数据格式
3. **数据导入**：运行导入脚本将数据写入数据库

### 3. 配置调整
根据不同环境和需求，可调整以下参数：
- 并行进程数（USER_DEFINED_CORES/MAX_WORKERS）
- 输入输出目录路径
- 数据库连接参数
- 数据过滤条件
